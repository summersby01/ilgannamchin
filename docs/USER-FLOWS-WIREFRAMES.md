# USER-FLOWS-WIREFRAMES.md

## 1. 사용자 흐름

### 1.1 기본 흐름
```text
홈 진입
↓
START 클릭
↓
게임 화면에서 사진 롤링
↓
STOP 클릭
↓
결과 화면 확인
↓
[사진 저장] 또는 [공유하기] 또는 [다시하기]
```

### 1.2 저장 흐름
```text
결과 화면
↓
사진 저장 클릭
↓
결과 카드 PNG 생성
↓
다운로드 성공 / 실패 안내
```

### 1.3 공유 흐름
```text
결과 화면
↓
공유하기 클릭
↓
지원 브라우저 확인
 ├─ 지원함 → Web Share API 실행
 └─ 지원 안 함 → 링크 복사
```

## 2. 와이어프레임

### 2.1 홈 화면

```text
┌────────────────────────┐
│                        │
│        일간남친        │

│   만나보세요           │
│                        │
│       [ START ]        │
│                        │
└────────────────────────┘
```

## 3. 모바일 우선 원칙
- 모든 주요 버튼은 하단 엄지 영역에서 누르기 쉬워야 한다.
- 텍스트 줄바꿈은 의도적으로 2~3줄 내로 제한한다.
- 결과 카드와 실제 페이지 배경은 분리해 저장 이미지가 깔끔해야 한다.

## 4. 향후 자동 롤링 구조 와이어프레임
프로젝트 메모:
- “게임화면구조 수정” 시 아래 구조로 변경

```text
홈 진입
↓
아이돌 롤링 자동 시작
↓
STOP 클릭
↓
결과 화면
```

이 구조에서는 홈 화면의 `START` 버튼이 제거되거나, 홈 진입 직후 곧바로 롤링 화면으로 이동하는 방식으로 바뀐다.
