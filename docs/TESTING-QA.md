# TESTING-QA.md

## 1. 테스트 범위
- 홈 화면 표시
- START 동작
- 롤링 동작
- STOP 동작
- 결과 표시
- 사진 저장
- 공유하기
- 다시하기
- 오류 처리
- 모바일 반응형

## 2. QA 체크리스트

### 2.1 홈 화면
- [ ] 앱 제목이 `일간남친`으로 보인다.
- [ ] 설명 문구가 2~3줄 안에서 자연스럽게 보인다.
- [ ] `START` 버튼이 잘 보인다.

### 2.2 게임 화면
- [ ] 이미지가 빠르게 바뀐다.
- [ ] 깜빡임이 과도하지 않다.
- [ ] `STOP` 버튼이 명확하다.

### 2.3 결과 화면
- [ ] 선택된 아이돌 이미지가 보인다.
- [ ] 이름이 정확히 표시된다.
- [ ] `사진 저장`, `공유하기`, `다시하기`가 모두 보인다.

### 2.4 저장 기능
- [ ] 저장된 PNG에 결과 카드만 포함된다.
- [ ] 텍스트가 잘리지 않는다.
- [ ] 이미지가 비어 있지 않다.
- [ ] iPhone Safari에서 최소 1회 테스트했다.

### 2.5 공유 기능
- [ ] 지원 환경에서 공유 시트가 열린다.
- [ ] 미지원 환경에서 링크 복사 fallback이 된다.

### 2.6 다시하기
- [ ] 상태가 초기화된다.
- [ ] 이전 결과가 남지 않는다.

## 3. 에러 시나리오
- 아이돌 목록이 비어 있음
- 이미지 URL 깨짐
- Supabase 응답 실패
- 저장 라이브러리 실패
- 공유 미지원

## 4. 테스트 기기 권장
- iPhone Safari
- Android Chrome
- 데스크톱 Chrome
- 데스크톱 Safari 가능 시 추가

## 5. 수동 테스트 케이스 예시
| ID | 시나리오 | 기대 결과 |
|---|---|---|
| TC-01 | 홈 진입 | 제목/설명/START 표시 |
| TC-02 | START 클릭 | 롤링 시작 |
| TC-03 | STOP 클릭 | 결과 화면 이동 |
| TC-04 | 사진 저장 클릭 | PNG 저장 |
| TC-05 | 공유하기 클릭 | 공유 또는 링크 복사 |
| TC-06 | 다시하기 클릭 | 초기 상태 복귀 |
| TC-07 | 아이돌 0명 | 오류 메시지 표시 |

## 6. 배포 전 최종 점검
- [ ] 환경변수 등록 확인

- [ ] 결과 저장 확인
- [ ] 잘못된 이미지 없는지 확인
