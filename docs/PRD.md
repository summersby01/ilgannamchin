# PRD — 일간남친

## 1. 제품 개요

### 1.1 제품명
**일간남친**

### 1.2 제품 설명
일간남친은 사용자가 가볍게 즐길 수 있는 팬게임 웹앱이다.  
게임을 시작하면 여러 남자 아이돌 사진이 빠르게 변경되고, 사용자가 `STOP` 버튼을 누르는 순간 멈춘 인물이 “오늘의 남자친구” 결과로 출력된다.

### 1.3 목적
- 팬들이 장난스럽게 즐길 수 있는 짧은 체류형 콘텐츠 제공
- 결과 이미지를 저장하거나 SNS에 공유하도록 유도
- 모바일에서 즉시 플레이 가능한 밈형 웹앱 구현

### 1.4 타깃 사용자
- K-pop 팬
- 팬덤 커뮤니티, X(트위터), 오픈채팅 사용자
- 짧고 즉각적인 리액션형 콘텐츠를 선호하는 사용자

## 2. 문제 정의
일반적인 팬놀이 콘텐츠는 공유는 쉽지만 직접 상호작용하는 재미가 약하다.  
일간남친은 사용자의 개입(`STOP`)을 넣어 결과에 “내가 고른 것 같은 느낌”을 주고, 결과 저장/공유를 통해 재방문과 확산을 노린다.

## 3. 성공 기준
### 핵심 지표
- 결과 화면 도달률
- 결과 저장 버튼 클릭률
- 공유 버튼 클릭률
- 1인당 평균 플레이 수
- 모바일 이탈률

### MVP 목표
- 사용자가 첫 방문 후 10초 이내에 게임 규칙을 이해할 수 있어야 한다.
- 결과 이미지를 저장할 수 있어야 한다.
- 모바일에서 깨지지 않아야 한다.

## 4. 핵심 사용자 흐름
```text
홈
↓
START
↓
게임 화면(이미지 롤링)
↓
STOP
↓
결과 화면
↓
[사진 저장] / [공유하기] / [다시하기]
```

## 5. 기능 요구사항

### 5.1 홈 화면
#### 목적
사용자가 게임을 시작하도록 유도한다.

#### 구성
- 앱 제목: `일간남친`
- 설명 문구:
  - `START를 눌러`
  - `오늘의 남자친구를 만나보세요`
- CTA 버튼: `START`

#### 요구사항
- 첫 화면에서 게임 방식이 직관적으로 이해되어야 한다.
- 모바일 중앙 정렬 레이아웃을 기본으로 한다.

---

### 5.2 게임 화면
#### 목적
랜덤 결과를 기대하게 만드는 롤링 애니메이션을 제공한다.

#### 구성
- 중앙 이미지 영역
- 보조 설명 문구:
  - `STOP을 눌러`
  - `오늘의 남자친구를 결정하세요`
- 주요 버튼: `STOP`

#### 요구사항
- 이미지 전환 속도는 80~150ms 범위에서 조정 가능해야 한다.
- 깜빡임이 심하지 않아야 한다.
- 로딩이 늦는 이미지는 사전 preload를 고려한다.

---

### 5.3 결과 화면
#### 목적
선택된 아이돌을 명확히 보여주고 저장·공유를 유도한다.

#### 구성
- 결과 타이틀: `오늘의 남자친구`
- 선택된 인물 사진
- 결과 문구: `{이름} 입니다 💘`
- 버튼:
  - `사진 저장`
  - `공유하기`
  - `다시하기`

#### 요구사항
- 결과 카드 자체가 저장 가능한 단일 시각 요소여야 한다.
- 공유용 텍스트는 별도 커스터마이즈 가능해야 한다.

---

### 5.4 사진 저장
#### 목적
사용자가 결과 화면을 이미지로 저장할 수 있도록 한다.

#### 요구사항
- 결과 카드 DOM을 PNG로 저장한다.
- 저장 대상은 결과 카드만 포함해야 한다.
- 저장 실패 시 안내 메시지를 보여준다.

---

### 5.5 공유하기
#### 목적
SNS 또는 메신저로 링크와 결과를 퍼뜨릴 수 있도록 한다.

#### 요구사항
- 모바일: Web Share API 우선 사용
- 미지원 환경: 링크 복사 fallback
- 공유 문구 예시:
  - `내 오늘의 남자친구 결과 나옴`
  - `{이름}이 나왔어`
  - `너도 해봐`

## 6. 비기능 요구사항
- 모바일 우선 반응형
- 첫 화면 LCP 최소화
- 결과 저장 기능은 주요 최신 모바일 브라우저에서 동작해야 함
- 접근성: 버튼은 명확한 라벨 필요
- 에러 시 빈 화면이 아닌 안내 문구 표시

## 10. 구조 변경용 내부 메모
현재 기본 구조:
```text
홈 → START → 롤링 → STOP → 결과
```

향후 사용 가능 구조:
```text
홈 → 자동 롤링 → STOP → 결과
```

프로젝트 내부에서 **“게임화면구조 수정”**이라고 말하면 위 자동 롤링 구조로 바꾸는 작업을 의미한다.
