# SKILL.md — Codex 작업 지침서

이 문서는 **일간남친** 프로젝트를 AI 코딩 에이전트(Codex 등)에게 안정적으로 지시하기 위한 작업 규칙 문서다.

## 1. 프로젝트 요약
- 프로젝트명: 일간남친
- 유형: 모바일 우선 팬게임 웹앱
- 핵심 기능:
  - 홈 화면
  - START 버튼
  - 아이돌 이미지 롤링
  - STOP 버튼
  - 결과 화면
  - 결과 이미지 저장
  - 공유하기
- 스택:
  - Node.js
  - Next.js
  - TypeScript
  - Tailwind CSS
  - Supabase
  - Vercel

## 2. 절대 유지해야 할 UX 규칙
### 기본 구조
```text
홈
↓
START
↓
롤링 화면
↓
STOP
↓
결과 화면
```

### 홈 화면 문구
- 제목: `일간남친`
- 설명:
  - `START를 눌러`
  - `오늘의 남자친구를 만나보세요`

### 게임 화면 문구
- 설명:
  - `STOP을 눌러`
  - `오늘의 남자친구를 결정하세요`

### 결과 화면 문구
- 제목: `오늘의 남자친구`
- 결과: `{이름} 입니다 💘`

## 3. 디자인 스킬 지침
Codex는 UI를 수정할 때 아래 원칙을 따라야 한다.

### 3.1 전체 톤
- 모바일 퍼스트
- 귀엽고 가볍지만 과도하게 유치하지 않음
- 팬덤 밈 콘텐츠 느낌
- 결과 카드가 공유 이미지로도 자연스러워야 함

### 3.2 레이아웃
- 화면 중앙 정렬
- 카드형 UI
- 버튼 간격 충분히 확보
- 모바일에서 텍스트가 지저분해 보이지 않도록 줄 수를 최소화

### 3.3 타이포
- 제목은 짧고 굵게
- 설명 문구는 2줄 내외
- 버튼 텍스트는 명확하게

### 3.4 결과 카드 디자인
- 저장 시 결과 카드만 이미지화
- 버튼, 배경, 부가 설명이 저장 이미지에 섞이지 않게 분리
- 사진, 이름, 제목의 위계가 분명해야 함

### 3.5 애니메이션
- 롤링은 빠르되 멀미를 유발하지 않게
- 불필요한 과한 bounce 금지
- STOP 후 결과는 즉시 안정적으로 보여야 함

## 4. 기술 규칙
- TypeScript 타입을 명확히 유지
- 컴포넌트 분리:
  - `HomeScreen`
  - `RollingScreen`
  - `ResultScreen`
  - `ResultCard`
- 저장 기능은 `html-to-image` 우선 사용
- 데이터는 Supabase에서 조회
- 결과 카드 저장 대상은 별도 `ref`로 관리

## 5. 코드 수정 원칙
- 기존 UX 문구를 임의로 바꾸지 말 것
- 버튼 역할을 혼동하지 말 것
  - START = 게임 시작
  - STOP = 랜덤 확정
- 상태 전환 구조를 명확히 유지할 것
- 모바일 화면에서 텍스트 겹침 방지 우선

## 6. 내부 약속어
### “게임화면구조 수정”
이 표현이 나오면 아래 변경을 의미한다.

#### 기존
```text
홈 → START → 롤링 → STOP → 결과
```

#### 변경 후
```text
홈 → 자동 롤링 → STOP → 결과
```

#### Codex가 해야 할 일
- 홈 화면에서 START 제거 또는 축소
- 진입 직후 자동 롤링 시작
- 필요하면 홈 화면 없이 곧바로 게임 화면으로 이동
- 관련 문구 수정
- phase 전환 로직 업데이트
- 불필요해진 START 관련 코드 정리

## 7. 수정 요청 응답 형식 권장
Codex는 변경 작업 시 아래 순서로 응답하는 것이 좋다.
1. 무엇을 바꾸는지 요약
2. 변경 파일 목록
3. 핵심 로직 설명
4. 주의점
5. 테스트 방법
